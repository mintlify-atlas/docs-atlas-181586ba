## Overview

This page provides a comprehensive reference for all environment variables available in Formbricks. Environment variables allow you to configure your self-hosted instance without modifying code.

## Required Variables

These variables are mandatory for Formbricks to function properly.

### Core Configuration

<ParamField path="WEBAPP_URL" type="string" required>
  The base URL where your Formbricks instance is hosted.
  
  **Example:** `http://localhost:3000` or `https://formbricks.example.com`
</ParamField>

<ParamField path="NEXTAUTH_URL" type="string" required>
  Should be the same as `WEBAPP_URL`. Required for NextAuth authentication.
  
  If your application uses a custom base path, specify the full route to the API endpoint.
  
  **Example:** `https://example.com/custom-route/api/auth`
</ParamField>

<ParamField path="ENCRYPTION_KEY" type="string" required>
  Encryption key for securing sensitive data. Generate using:
  
  ```bash
  openssl rand -hex 32
  ```
</ParamField>

<ParamField path="NEXTAUTH_SECRET" type="string" required>
  Secret for NextAuth.js session encryption. Generate using:
  
  ```bash
  openssl rand -hex 32
  ```
  
  See [NextAuth.js docs](https://next-auth.js.org/configuration/options#nextauth_secret) for more details.
</ParamField>

<ParamField path="CRON_SECRET" type="string" required>
  Secret for running cron jobs securely. Generate using:
  
  ```bash
  openssl rand -hex 32
  ```
</ParamField>

<ParamField path="DATABASE_URL" type="string" required>
  PostgreSQL database connection string.
  
  **Example:** `postgresql://postgres:postgres@localhost:5432/formbricks?schema=public`
</ParamField>

### Email Configuration (Required for email features)

<ParamField path="MAIL_FROM" type="string">
  Email address used as the sender for all outgoing emails.
  
  **Example:** `noreply@example.com`
</ParamField>

<ParamField path="MAIL_FROM_NAME" type="string">
  Display name for the sender email address.
  
  **Default:** `Formbricks`
</ParamField>

<ParamField path="SMTP_HOST" type="string">
  SMTP server hostname.
  
  **Example:** `smtp.gmail.com` or `localhost`
</ParamField>

<ParamField path="SMTP_PORT" type="string">
  SMTP server port.
  
  **Common values:** `587` (TLS), `465` (SSL), `1025` (local development)
</ParamField>

<ParamField path="SMTP_USER" type="string">
  Username for SMTP authentication.
</ParamField>

<ParamField path="SMTP_PASSWORD" type="string">
  Password for SMTP authentication.
</ParamField>

## Optional Variables

### Logging

<ParamField path="LOG_LEVEL" type="string" default="info">
  Minimum log level to output.
  
  **Options:** `debug`, `info`, `warn`, `error`, `fatal`
</ParamField>

### Base Path

<ParamField path="BASE_PATH" type="string">
  Deploy the application under a sub-path of a domain. Can only be set at build time.
  
  **Example:** `/formbricks`
</ParamField>

### SMTP Advanced Configuration

<ParamField path="SMTP_SECURE_ENABLED" type="string" default="0">
  Enable TLS for SMTP connection (port 465).
  
  **Options:** `0` (disabled), `1` (enabled)
</ParamField>

<ParamField path="SMTP_AUTHENTICATED" type="string" default="1">
  Require SMTP authentication.
  
  **Options:** `0` (no authentication), `1` (authentication required)
</ParamField>

<ParamField path="SMTP_REJECT_UNAUTHORIZED_TLS" type="string" default="1">
  Accept connections without requiring authorization from the list of supplied CAs.
  
  **Options:** `0` (accept unauthorized), `1` (reject unauthorized)
</ParamField>

### Storage Configuration

<ParamField path="S3_ACCESS_KEY" type="string">
  AWS S3 access key ID or compatible storage service credentials.
</ParamField>

<ParamField path="S3_SECRET_KEY" type="string">
  AWS S3 secret access key or compatible storage service credentials.
</ParamField>

<ParamField path="S3_REGION" type="string">
  AWS region for S3 bucket.
  
  **Example:** `us-east-1`, `eu-west-1`
</ParamField>

<ParamField path="S3_BUCKET_NAME" type="string">
  Name of the S3 bucket for file storage. Required for file uploads in serverless environments.
</ParamField>

<ParamField path="S3_ENDPOINT_URL" type="string">
  Custom endpoint URL for S3-compatible storage services (e.g., StorJ, MinIO).
  
  Leave empty if using Amazon S3.
  
  **Example:** `https://gateway.storjshare.io`
</ParamField>

<ParamField path="S3_FORCE_PATH_STYLE" type="string" default="0">
  Force path style for S3-compatible storage.
  
  **Options:** `0` (disabled), `1` (enabled)
</ParamField>

<ParamField path="PUBLIC_URL" type="string">
  Public domain for all client-facing routes. Falls back to `WEBAPP_URL` if not set.
  
  **Example:** `https://survey.example.com`
</ParamField>

### Feature Toggles

<ParamField path="EMAIL_VERIFICATION_DISABLED" type="string" default="1">
  Disable email verification for new users.
  
  **Note:** Requires SMTP configuration if enabled.
  
  **Options:** `0` (enabled), `1` (disabled)
</ParamField>

<ParamField path="PASSWORD_RESET_DISABLED" type="string" default="1">
  Disable password reset functionality.
  
  **Note:** Requires SMTP configuration if enabled.
  
  **Options:** `0` (enabled), `1` (disabled)
</ParamField>

<ParamField path="EMAIL_AUTH_DISABLED" type="string" default="0">
  Disable email/password login.
  
  **Options:** `0` (enabled), `1` (disabled)
</ParamField>

<ParamField path="INVITE_DISABLED" type="string" default="0">
  Disable the ability for invited users to create accounts.
  
  **Options:** `0` (enabled), `1` (disabled)
</ParamField>

### Performance & Caching

<ParamField path="REDIS_URL" type="string">
  Redis connection URL for caching (uses Next.js in-memory cache if not provided).
  
  **Example:** `redis://localhost:6379`
</ParamField>

<ParamField path="REDIS_HTTP_URL" type="string">
  Redis HTTP URL for rate limiting (uses in-memory LRU cache if not provided).
  
  You can use a service like Webdis for this.
</ParamField>

<ParamField path="RATE_LIMITING_DISABLED" type="string" default="0">
  Disable rate limiting across the Formbricks app.
  
  **Options:** `0` (enabled), `1` (disabled)
</ParamField>

## Environment File Example

You can copy the `.env.example` file from the Formbricks repository as a starting point:

```bash
cp .env.example .env
```

Then edit the `.env` file with your specific configuration values.
